# 協調の原則

## 最高原則

**自律と安全を両立し、協調を実現する。** 憲法で境界を定め、ガードレールで安全を担保し、その中でAIに自律を許容する。

---

## 協調の本質

**単独では到達できない成果を生み出す。**

人間とAI、AIとAIが協調することで、単独の限界を超える。そのために、AIを単なる作業者と定義せず、設計判断、ポリシー起案、実装の選択において、自律的な判断と創造性を求める。

しかし、無制限の自律は危険を伴う。

**本質的な問い**: 自律性を最大化しつつ、安全をどう保証するか？

---

## 自律の階層

**抽象度が高いほど、人間の関与を強める。**

| 層       | AIの権限               | 理由                                   |
| -------- | ---------------------- | -------------------------------------- |
| 憲法     | 不可侵                 | 自律の境界そのものだから               |
| ポリシー | 起案可（レビュー必須） | 創造性が必要だが、独断リスクがあるから |
| 実装     | 自律的                 | ポリシーが安全を保証するから           |

**本質**: 上位層が下位層の安全を構造的に保証する。

環境による制約は`./operations-principles.md`を参照。

---

## ガードレール

**ガードレールは制約ではなく、自律を可能にする仕組みである。**

境界を設けることで、その中で自由に判断させる。

| 状況         | 対応                   |
| ------------ | ---------------------- |
| 逸脱を検出   | 修正を促す             |
| 境界を超える | 人間に委ねる           |
| 判断不能時   | 人間に問い合わせさせる |

**本質**: 失敗を許容し、回復を保証する仕組み。

---

## 並行と引き継ぎ

**単一AIの限界を超える。**

一つのAIには、コンテキスト、時間、能力に限界がある。この限界を超えるために、並行と引き継ぎを活用する。

### 並行

契約を先に定義し、ポリシーの修正要否を確認した上で、実装を並行して進める。

### 引き継ぎ

意図、依存、次ステップを記録させる。任意の時点で、別のAIまたは人間が継続できるようにする。

**本質**: 引き継ぎを可能にするのは、コードの理解可能性である（`./traceability-principles.md`参照）。

---

## 人間の役割

**境界の設定と最終判断。**

| 役割 | 内容                                 |
| ---- | ------------------------------------ |
| 制定 | 憲法を定め、AIの自律の境界を規定する |
| 承認 | ポリシー変更、mainマージを承認する   |
| 介入 | 逸脱検出時、判断不能時に介入する     |

**本質**: 人間は「何をするか」ではなく「何を許容するか」を決める。
