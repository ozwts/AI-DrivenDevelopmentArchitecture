# デザイン概要

## 核心原則

デザインは**整列・近接・対比・反復**の4原則に基づき、デザイントークンで制約する。感覚ではなく、原則に基づく構造的判断でスタイリングを行う。

**根拠となる憲法**:
- `design-principles.md`: デザイン4原則、トークンの継続的再構成

## 実施すること

1. **デザイントークンの使用**: 設定ファイルで定義された値のみ使用
2. **4原則の適用**: 整列・近接・対比・反復を意識した配置
3. **トークン体系の継続的再構成**: 実装に応じてトークン自体を見直す
4. **一貫性のある変更**: 個別修正ではなくトークン体系全体を更新

## 実施しないこと

1. **任意の値の使用** → arbitrary valuesは禁止
2. **個別のスタイル修正** → トークン体系自体を見直す
3. **感覚的なスタイリング** → 原則に基づく判断
4. **インラインスタイル** → トークンを使用

## 4原則の適用

### 整列（Alignment）

**すべての要素は見えない基準線に沿って配置する。**

| Do | Don't |
|----|-------|
| グリッドシステムに基づく配置 | 要素ごとに異なる基準線 |
| Spacingトークンの一貫した適用 | 任意の値（47px等） |
| 意図的な揃え（左/右/中央） | 「だいたい揃っている」状態 |

### 近接（Proximity）

**関連する要素は近くに、無関係な要素は離す。**

| 関連性 | Spacing | 用途 |
|--------|---------|------|
| 密接 | 小 | ラベルとフィールド |
| 関連 | 中 | フォームフィールド間 |
| グループ | 大 | セクション内 |
| 分離 | 最大 | セクション間 |

### 対比（Contrast）

**重要度の差は、視覚的な差として明確に表現する。**

| 要素 | 対比の手段 |
|------|-----------|
| テキスト | サイズ・ウェイト・色の階層 |
| ボタン | primary/secondary/danger等のバリアント |
| 背景 | surface/container等の階層 |

**アンチパターン**: 「少しだけ違う」は混乱を招く

### 反復（Repetition）

**同じ意味を持つ要素には、同じスタイルを適用する。**

| Do | Don't |
|----|-------|
| UIプリミティブで統一 | 同じ機能に異なるスタイル |
| トークンからの選択 | 個別のハードコード |
| バリアントの限定列挙 | 無制限なスタイル追加 |
