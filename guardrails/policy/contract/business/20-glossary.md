# 用語集（Glossary）ポリシー

## 目的

用語集は、ビジネス全体で共有される語彙を定義する。命名の源泉となる。

## 配置

```
/contracts
  /business
    glossary.md
```

## 記述形式

```markdown
# 用語集

| 用語 | 英語名 | 定義 |
|------|--------|------|
| {用語} | {English} | {定義} |
```

## 例

```markdown
# 用語集

| 用語 | 英語名 | 定義 |
|------|--------|------|
| ユーザー | User | アプリケーションを利用する人。認証済みのアカウントを持つ |
| TODO | Todo | やるべきタスク。タイトル、説明、ステータス、優先度、期限、担当者を持つ |
| プロジェクト | Project | TODOをグループ化するための分類。名前、説明、カラーを持つ |
```

## 原則

| 原則 | 説明 |
|------|------|
| 一意性 | 同じ概念に複数の名前を与えない |
| 明確性 | 曖昧な定義を避ける |
| ビジネス視点 | 技術的な実装詳細は書かない |
