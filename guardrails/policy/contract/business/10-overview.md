# ビジネス契約ポリシー

## 目的

ビジネス契約は、ユビキタス言語の源泉である。ドメインの語彙、構造、振る舞いを定義し、コードの導出元となる。

## 構造

```
/contracts
  /business                      # ビジネス契約
    glossary.md                  # ビジネス用語集
    /{domain}                    # ドメイン単位
      definition.md              # 定義（概念・属性・制約）
      /scenario                  # シナリオ（振る舞い）
        {scenario}.md
  /api                           # API契約
    /{domain}
  /db                            # DB契約
    /{domain}
```

## 2つの源泉

| 源泉 | 内容 | 導出されるもの |
|------|------|---------------|
| definition | 概念・属性・制約 | ドメインモデル・値オブジェクト |
| scenario | 振る舞い・シナリオ | ユースケース・アプリケーションサービス |

## ドメインモデルカット

| 原則 | 理由 |
|------|------|
| ドメイン単位で凝集 | 変更が1箇所で完結する |
| 境界が明確 | 他ドメインへの波及を防ぐ |
| 独立して進化 | ドメインごとに異なるペースで発展できる |

## コードとの一致

- コードは契約から導出する
- 契約にない概念をコードに導入しない
- コードの変更は契約の変更を伴う
