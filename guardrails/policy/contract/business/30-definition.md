# 定義（Definition）ポリシー

## 目的

定義は、ドメインの概念・属性・制約を記述する。ドメインモデルの導出元となる。

## 配置

```
/contracts
  /business
    /{domain}
      definition.md
```

## 記述形式

```markdown
# {概念名}

## 定義

{この概念が何であるか}

## 属性

- {属性名}：{説明}

## 制約

- {制約の説明}
```

## 例

```markdown
# 契約

## 定義

顧客とサービス提供者間の合意。開始日から終了日までの期間、サービスを提供する約束。

## 属性

- 状態：申込中、有効、終了、解約のいずれか
- 期間：開始日と終了日
- 顧客：契約の当事者

## 制約

- 契約は常に1つの状態を持つ
- 有効な契約には開始日が必須
- 開始日は終了日より前

# 契約期間

## 定義

契約の有効期間を表す。

## 属性

- 開始日：契約が有効になる日
- 終了日：契約が終了する日

## 制約

- 開始日は終了日より前
```

## 原則

| 原則 | 説明 |
|------|------|
| 用語集と一致 | 定義内の用語は用語集と完全に一致させる |
| 1ドメイン1ファイル | ドメイン単位で凝集させる |
| ビジネス視点 | 技術的な型や実装詳細は書かない |
| 属性と制約を併記 | 構造と不変条件を一体として定義 |
